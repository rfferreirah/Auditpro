Guia de Deploy na Nuvem (Render / Railway)

Este guia explica como colocar sua aplicação no ar gratuitamente (ou com baixo custo) usando Render ou Railway. Ambos são excelentes opções que se conectam direto ao seu GitHub.

Pré-requisitos

  1. Ter o código do projeto em um repositório no GitHub.
  2. Ter o arquivo Procfile na raiz do projeto (este arquivo define como a nuvem inicia seu app).
  3. Ter o requirements.txt atualizado com a biblioteca gunicorn.

Opção 1: Deploy no Render (Plano Gratuito)

O Render oferece um plano gratuito ("Free Tier") excelente para projetos pessoais.

  1. Crie uma conta em render.com.
  2. Clique em "New +" e selecione "Web Service".
  3. Conecte sua conta do GitHub e selecione o repositório deste projeto.
  4. Preencha os campos de configuração:
    • Name: audit-pro (ou o nome que preferir)
    • Region: Escolha a mais próxima (ex: Ohio ou Frankfurt)
    • Branch: main
    • Runtime: Python 3
    • Build Command: pip install -r requirements.txt
    • Start Command: gunicorn web_app:app
  5. Configuração de Ambiente (Environment Variables):

    Role até a seção "Environment Variables" e adicione as chaves:

    • REDCAP_API_URL: (Sua URL do REDCap)
    • REDCAP_API_TOKEN: (Seu Token)
    • PYTHON_VERSION: 3.10.0 (Recomendado)
  6. Clique em "Create Web Service".
Sucesso: O deploy levará alguns minutos. Ao final, o Render fornecerá uma URL pública (ex: https://audit-pro.onrender.com).

Opção 2: Deploy no Railway

O Railway é conhecido pela simplicidade e robustez. Geralmente oferece um período de teste ou créditos iniciais.

  1. Crie uma conta em railway.app.
  2. Clique em "New Project" -> "Deploy from GitHub repo".
  3. Selecione o repositório do projeto.
  4. Clique no projeto criado e navegue até a aba "Variables".
  5. Adicione as variáveis do seu arquivo .env:
    • REDCAP_API_URL
    • REDCAP_API_TOKEN
  6. Vá em "Settings" -> "Generate Domain" para criar uma URL pública acessível.
  7. O Railway detectará automaticamente o arquivo Procfile e iniciará o deploy.

Dicas Importantes para Nuvem

Segurança (HTTPS e Dados):
  • Tanto Render quanto Railway configuram HTTPS automaticamente.
  • NUNCA suba seu arquivo .env para o GitHub. Use sempre o painel da plataforma para configurar as variáveis de ambiente.

Sobre Arquivos (PDFs): Em planos gratuitos ou arquiteturas modernas de nuvem, o sistema de arquivos é "efêmero". Isso significa que arquivos criados na pasta outputs/ podem desaparecer se o servidor reiniciar. Para este projeto (gerar e baixar PDF na hora), isso não é um problema. Se precisar de armazenamento permanente, considere usar um serviço como AWS S3 ou Google Cloud Storage.
