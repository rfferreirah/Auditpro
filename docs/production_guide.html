Guia de Preparação para Produção

Este guia descreve os passos para rodar o REDCap Data Quality Intelligence Agent em ambiente de produção (Server ou Windows local robusto) e como executar os testes.

1. Instalação de Dependências

Certifique-se de que todas as dependências estão instaladas, incluindo as ferramentas de produção (waitress) e teste (pytest) adicionadas recentemente.

pip install -r requirements.txt

2. Configuração (.env)

Verifique o arquivo .env antes de subir para produção:

  • DEBUG: Deve estar como false.
  • REDCAP_API_TOKEN: Se for rodar em servidor compartilhado, considere usar variáveis de ambiente do sistema em vez do arquivo .env para segurança.
  • ANTHROPIC_API_KEY: Garanta que a chave é válida.
# Exemplo .env para Produção
DEBUG=false
REDCAP_TIMEOUT=60

3. Executando Testes

Para garantir que tudo está funcionando antes de iniciar o servidor, execute os testes automatizados.

# Rodar todos os testes
python -m pytest

# Rodar apenas testes básicos (smoke tests)
python -m pytest tests/test_smoke.py

Se todos os testes passarem (aparecer verde), a aplicação está estável para iniciar.

4. Rodando em Produção (Windows/Linux)

Não utilize python web_app.py em produção (o servidor de desenvolvimento do Flask não é seguro ou performático). Use o servidor Waitress que foi instalado.

Comando para Iniciar (Waitress)

Crie um arquivo .bat (Windows) ou script shell com o seguinte comando:

waitress-serve --listen=*:5000 wsgi:app
  • --listen=*:5000: Escuta na porta 5000 em todas as interfaces de rede (acessível por outros computadores na rede).
  • wsgi:app: Aponta para o arquivo wsgi.py que criamos.

Executando em Background (Windows)

Para rodar sem manter o terminal aberto, você pode usar o "Gerenciador de Tarefas" ou configurar como um Serviço do Windows (usando ferramentas como NSSM).

5. Checklist Final

  • DEBUG=false no .env
  • Dependências instaladas (pip install -r requirements.txt)
  • Testes passando (pytest)
  • Logs sensíveis removidos ou protegidos
  • Servidor rodando via Waitress (não Flask dev server)
